# Four Pillars of Value — crosswalk (process + intent → pillar)

A shared vocabulary so every team articulates value in a similar way. Single source of truth for
mapping **any** user's work to the four value pillars. Both `map-my-work` and `cowork-roi-report` read
this file — do not fork the mapping elsewhere. Nothing here is specific to any individual.

## The four pillars

| Pillar | Type | Definition | Example KPI |
|---|---|---|---|
| **Revenue Growth** | Tangible · money coming in | Drive measurable increases in top-line performance — demand created, converted, and monetised (new opportunities, win rates, pricing, faster deal cycles). | Incremental gross revenue |
| **Cost Reduction** | Tangible · money going out | Reduce the cost base while maintaining or improving output — eliminate inefficiencies, automate manual work, optimise spend; value via direct savings or capacity redeployed. | Labor & budget savings |
| **Risk Mitigation** | Intangible · money going out | Protect enterprise value by reducing financial, operational, and compliance risk — detect issues earlier, improve controls, enable faster correction. | Penalties & losses avoided |
| **Transformation** | Intangible · money coming in | Enable new ways of working — better/faster decisions, more responsive operations, stronger collaboration; increase adoption of AI-assisted workflows and workforce capability. | Adoption, decision quality, retention |

**Rule of precedence:** `pillar = intent-verb rule when it clearly fires; otherwise the process default.`
A process is **not** locked to one pillar — the same process can land in different pillars across
sessions based on intent.

## 1. Process default pillar (set when you derive each user's processes)

There is no fixed process list — `map-my-work` derives **each user's own** processes. Give every
derived process a **default pillar** from its dominant purpose:

| If the process mostly… | Default pillar |
|---|---|
| **wins / grows / converts** customers, demand, or top-line | Revenue Growth |
| **reduces / allocates / optimises cost** (spend, budget, headcount, chargeback) | Cost Reduction |
| **protects against risk** (compliance, controls, security, audit) | Risk Mitigation |
| **changes how work is done** (new capability, automation, decisions, adoption) | Transformation |
| doesn't clearly fit | default to **Transformation** (productivity/new-ways-of-working) |

## 2. Intent-verb → pillar (overrides the default when the signal is clear)

| If the work's intent / JTBD is… | Pillar |
|---|---|
| **grow / win / convert / monetise** demand, deals, or top-line | Revenue Growth |
| **reduce / allocate / avoid / control / forecast** spend, cost, or budget | Cost Reduction |
| **protect / prevent / detect / comply / control** risk, exposure, or loss | Risk Mitigation |
| **build / automate / scale / improve adoption / speed decisions / enhance capability** | Transformation |

## 3. Assignment rules

1. Assign **exactly one** pillar per workflow / session.
2. If an intent signal (§2) clearly fires, it **wins** over the process default (§1).
3. If no intent signal fires, use the process default (§1).
4. If nothing fits, default to **Transformation** — never force-fit a tangible pillar (Revenue / Cost)
   onto work with no money signal.
5. Pillars with no qualifying work in the window render as **zero / empty**, not hidden — an empty
   band is a real signal, not a gap to paper over.
